# 🗺️ Project Roadmap: evangelho-do-dia

The strategic evolution of the **evangelho-do-dia** ecosystem, focused on scalability and depth.

## 🏁 Phase 1: Foundation (Short-Term)
- [x] Initial scaffolding with Expo and Node.js.
- [x] Functional Scraper for Liturgia Diária.
- [x] MVP Telegram Bot for daily gospel delivery.
- [ ] Unit tests for the scraping logic (Vitest).

## 🏔️ Phase 2: Refinement (Medium-Term)
- [ ] **Mobile Overhaul**: Implement Shadcn/ui-inspired components for React Native.
- [ ] **Caching Layer**: Local persistence for offline reading using SQLite or AsyncStorage.
- [ ] **Multi-source Logic**: Support for multiple liturgical calendars and sources.
- [ ] **Interactive Bot**: Add commands for saint-of-the-day and audio reflections.

## 🌌 Phase 3: Expansion (Long-Term)
- [ ] **AI Reflection Engine**: Generate context-aware spiritual reflections using Gemini API.
- [ ] **Audio-Streaming**: Integration with EdgeTTS for automated audio versions of the gospel.
- [ ] **Social Sync**: Automated posting to Instagram/Twitter daily highlights.
- [ ] **Community API**: Public REST API for external spiritual application integrations.
